## Part 1. 프로그래밍기능사 실기 (시나공)

### 1-1. 라이브러리 (Library)

- **정의**: 자주 쓰는 함수·클래스를 미리 만들어 모아둔 것. 개발자가 매번 바닥부터
  구현하지 않고 가져다 씀.
- **표준 라이브러리**: 프로그래밍 언어에 기본으로 포함된 라이브러리
  (별도 설치 없이 바로 사용)
- **외부 라이브러리**: 별도로 설치해서 사용하는 라이브러리

**언어별 대표 예시**
| 언어 | 표준 라이브러리 예 |
|---|---|
| C | stdio.h(입출력), stdlib.h(메모리/변환), string.h(문자열), math.h(수학) |
| Java | java.lang(기본), java.util(자료구조), java.io(입출력), java.sql(DB) |
| Python | 내장 함수 + import로 불러오는 모듈 (math, random, datetime 등) |

**시험 포인트**: 어떤 기능이 어느 헤더/패키지에 속하는지 매칭하는 문제가 나온다.
(예: `printf` → stdio.h / `malloc` → stdlib.h)

---

### 1-2. 표준 (Standard)

- **코딩 표준 / 코딩 컨벤션**: 여러 사람이 같은 스타일로 코드를 작성하도록 정한 규칙.
  - 목적: **가독성, 유지보수성, 협업 효율** 향상
  - 예: 들여쓰기, 네이밍 규칙(카멜케이스/스네이크케이스), 주석 작성법
- **표준화의 이유**: 코드를 "내가 읽기 좋게"가 아니라 **"팀이 읽기 좋게"** 만드는 것.

> CoFit 개발하면서 계속 지켜온 것과 같은 맥락 —
> 네이밍 일관성, 계층별 역할 분리, DTO/Entity 구분 등이 다 "표준"의 실천.

### 1-3. 객체지향 프로그래밍 (OOP)

**4대 특징 — 시험 단골**

| 특징 | 의미 |
|---|---|
| **캡슐화(Encapsulation)** | 데이터와 기능을 하나로 묶고, 내부를 숨김. 외부는 메서드로만 접근 |
| **상속(Inheritance)** | 부모 클래스의 속성·기능을 자식이 물려받음. 코드 재사용 |
| **다형성(Polymorphism)** | 같은 이름의 메서드가 상황에 따라 다르게 동작 (오버로딩/오버라이딩) |
| **추상화(Abstraction)** | 공통 속성·기능을 뽑아 정의. 불필요한 세부는 감춤 |

**오버로딩 vs 오버라이딩** (헷갈리기 쉬움 — 시험 출제)
- **오버로딩(Overloading)**: 같은 클래스 내, **같은 이름 다른 매개변수**의 메서드 여러 개
- **오버라이딩(Overriding)**: 부모의 메서드를 **자식이 재정의**

**클래스 / 객체 / 인스턴스**
- 클래스 = 설계도
- 객체 = 설계도로 만들 수 있는 대상
- 인스턴스 = 실제로 메모리에 생성된 객체

**접근 제어자**
| 제어자 | 접근 범위 |
|---|---|
| private | 같은 클래스 내에서만 |
| default | 같은 패키지 내 |
| protected | 같은 패키지 + 상속받은 자식 |
| public | 어디서나 |

> 실무 연결: CoFit Entity에서 `@NoArgsConstructor(access = PROTECTED)`로
> 기본 생성자를 막고, setter 대신 의도가 드러나는 메서드만 public으로 연 것 =
> **캡슐화의 실제 적용**. 이론이 실무와 붙는 지점.

## Part 2. Microsoft Dynamics 365 F&O — 모듈 1: 내비게이션

> 입사 전 선행 학습. MS 공식 트레이닝 모듈 1 완료 + 모듈 테스트 통과.

### 2-1. 핵심 UI 구성 요소

| 요소 | 역할 |
|---|---|
| **Navigation pane** (탐색 창) | 왼쪽 메뉴. 모듈·기능으로 이동하는 주 경로 |
| **Navigation bar** (탐색 표시줄) | 상단 바. 검색, 설정, 알림 등 |
| **Workspaces** (작업 영역) | 특정 역할·업무에 필요한 정보를 한 화면에 모아둔 대시보드 |
| **Search** (검색) | 페이지·기능을 이름으로 빠르게 찾음 |
| **Action pane** (작업 창) | 페이지 상단의 버튼 모음. 현재 화면에서 할 수 있는 동작 |
| **FastTabs** | 폼 안에서 정보를 접었다 펼 수 있는 섹션 |

### 2-2. 이해한 포인트

- **Workspace가 핵심 개념**: 단순 메뉴가 아니라, **역할 기반(role-based)으로
  필요한 데이터·작업·인사이트를 한곳에 묶은 화면.**
  예) 구매 담당자용 워크스페이스에는 승인 대기 발주, 입고 예정 등이 모여 있음.
- **검색(Search)**: 메뉴 구조를 다 외우지 않아도 페이지 이름으로 바로 이동 가능.
  실무에서 메뉴 경로를 헤매지 않는 게 곧 속도.
- **Action pane**: "지금 이 화면에서 뭘 할 수 있는가"가 여기 모여 있음.
  → 화면마다 가능한 동작이 달라진다.
