# 2026-06-10 TIL — Array, Tree & Sorting

## 배열 (Array)

- 같은 타입의 데이터를 연속된 메모리 공간에 저장하는 자료구조
- 인덱스(index)로 요소에 직접 접근 → 탐색 속도 빠름 O(1)
- 삽입/삭제는 느림 O(n) → 요소를 밀거나 당겨야 하기 때문
- 1차원 배열, 2차원 배열(행렬) 구조 존재

```
예시:
int[] arr = {10, 20, 30, 40, 50};
arr[0] = 10, arr[2] = 30
```

## 트리 (Tree)

### 기본 개념
- 노드(Node)와 간선(Edge)으로 구성된 계층적 비선형 자료구조
- 사이클 없음, 루트 노드는 1개

### 주요 용어
| 용어 | 설명 |
|------|------|
| 루트(Root) | 최상위 노드 |
| 부모(Parent) | 상위 노드 |
| 자식(Child) | 하위 노드 |
| 리프(Leaf) | 자식 없는 최하위 노드 |
| 깊이(Depth) | 루트부터 해당 노드까지 거리 |
| 차수(Degree) | 노드의 자식 수 |

### 이진 트리 (Binary Tree)
- 자식 노드 최대 2개
- 이진 탐색 트리(BST): 왼쪽 < 부모 < 오른쪽

### 트리 순회 3가지
| 순회 | 순서 | 암기 |
|------|------|------|
| 전위(Preorder) | 루트→왼쪽→오른쪽 | 루왼오 |
| 중위(Inorder) | 왼쪽→루트→오른쪽 | 왼루오 |
| 후위(Postorder) | 왼쪽→오른쪽→루트 | 왼오루 |

## 정렬 (Sorting)

| 정렬 | 시간복잡도 | 특징 |
|------|-----------|------|
| 버블 정렬 | O(n²) | 인접 두 원소 비교/교환 |
| 선택 정렬 | O(n²) | 최솟값 찾아 맨 앞으로 |
| 삽입 정렬 | O(n²) | 적절한 위치에 삽입 |
| 퀵 정렬 | O(n log n) | 피벗 기준 분할 |
| 병합 정렬 | O(n log n) | 분할 후 합병, 안정 정렬 |
| 힙 정렬 | O(n log n) | 힙 자료구조 활용 |

```
O(n²)      → 버블, 선택, 삽입
O(n log n) → 퀵, 병합, 힙
```

---

## 오늘 핵심
- 배열: 인덱스 접근 O(1), 삽입/삭제 O(n)
- 트리 순회: 전위(루왼오) / 중위(왼루오) / 후위(왼오루)
- 정렬: O(n²) vs O(n log n) 구분 필수
