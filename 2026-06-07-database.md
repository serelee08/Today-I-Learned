# 2026-06-07 TIL — Database & ERD

## 관계형 데이터베이스 (Relational Database)
- 데이터를 테이블(행/열) 형태로 저장
- 테이블 간 관계를 PK/FK로 연결
- 대표적인 RDBMS: Oracle, MySQL, MariaDB, PostgreSQL

## 키(Key) 종류
- PK (Primary Key): 테이블에서 행을 유일하게 식별하는 값. 중복 불가, NULL 불가
- FK (Foreign Key): 다른 테이블의 PK를 참조하는 컬럼. 테이블 간 관계 표현
- UK (Unique Key): 중복 불가, NULL 허용

## 정규화 (Normalization)
- 데이터 중복을 줄이고 일관성을 높이는 설계 원칙
- 1NF: 원자값만 저장 (반복 그룹 제거)
- 2NF: 부분 함수 종속 제거
- 3NF: 이행 함수 종속 제거

## ERD (Entity Relationship Diagram)
- 테이블 간의 관계를 시각화한 다이어그램
- 1:1 관계: 한 행이 다른 테이블의 한 행과 연결
- 1:N 관계: 한 행이 다른 테이블의 여러 행과 연결 (가장 많이 씀)
- N:M 관계: 중간 테이블(junction table)로 해결

## MariaDB vs Oracle
| 항목 | MariaDB | Oracle |
|------|---------|--------|
| 자동증가 | AUTO_INCREMENT | SEQUENCE |
| 날짜 | DATETIME | DATE |
| 문자열 | VARCHAR | VARCHAR2 |
