# 2026-06-26 운영체제(Operating System) 기초

## 운영체제란
- 컴퓨터 하드웨어와 사용자 사이에서 하드웨어를 관리하고, 사용자가 컴퓨터를 편리하게 사용하도록 돕는 시스템 소프트웨어
- 자원 관리자(Resource Manager) 역할: CPU, 메모리, 입출력장치, 파일 등을 관리

## 운영체제의 목적 (성능 평가 기준)
| 기준 | 의미 |
|------|------|
| 처리 능력 (Throughput) | 일정 시간 내 처리하는 작업량 → **높을수록 좋음** |
| 반환 시간 (Turnaround Time) | 작업 의뢰 후 결과를 받기까지의 시간 → **짧을수록 좋음** |
| 사용 가능도 (Availability) | 자원을 사용하고자 할 때 즉시 사용 가능한 정도 → **높을수록 좋음** |
| 신뢰도 (Reliability) | 주어진 문제를 정확하게 해결하는 정도 → **높을수록 좋음** |

## 운영체제 운영 방식 (발전 순서)
1. **일괄 처리 시스템 (Batch)** : 작업을 모아 한 번에 처리
2. **다중 프로그래밍 (Multi-programming)** : CPU 유휴 시간 동안 다른 작업 실행
3. **시분할 시스템 (Time Sharing)** : CPU 시간을 잘게 나눠 여러 사용자에게 할당 (= Round Robin)
4. **다중 처리 시스템 (Multi-processing)** : CPU 여러 개로 동시에 처리
5. **실시간 시스템 (Real Time)** : 정해진 시간 안에 반드시 처리 (예: 항공 예약, 미사일 제어)
6. **분산 처리 시스템 (Distributed)** : 여러 컴퓨터를 네트워크로 연결해 처리

## 운영체제의 구성
- **제어 프로그램 (Control Program)**
  - 감시 프로그램 (Supervisor)
  - 작업 관리 프로그램 (Job Management)
  - 데이터 관리 프로그램 (Data Management)
- **처리 프로그램 (Processing Program)**
  - 언어 번역 프로그램 (컴파일러, 어셈블러, 인터프리터)
  - 서비스 프로그램
  - 문제 처리 프로그램
